autoFuzz.py

Ive been having alot of fun with URLs and strings with web apps lately, so I made a tool to play with URLs in search of potentially interesting findings. I figured I could pull a proxy up and let my tool run all sorts of weird strings by the web app and see how it responds. Hopefully we can find some sort of neat error.

Requirements

I definitely couldnt do this without some help with the heavy lifting.
In order to run this you would need:

  • BeautifulSoup
  • requests
  • selenium

Which can all be easily gotten from pip.

You will also need the ‘Big List of Naughty Strings’ which you can get directly from their github.

GeckoDriver : Youll need to download the geckodriver for selenium and move it to /usr/bin/local/ in order to get the browser driver to work with selenium.

The tool at a glance

I havent had much time to use this extensively, I just finished getting it to a point where I felt comfortable pushing it to my git. However from what little use I have put it to, it was more than happy to furnish some mild results.

Pretty simple, we can pass it command line arguments since just running all functionality will take a while. Also worth nothing is that as it runs the fuzzing of the URL, it will display the page to you so you can see if any error is spit out. In the future we will work on being able to pass lists of URLs and piping our output without the need for a browser displaying it.

In the picture above we have ran the Bad 404 function, looking to see if we can inject text into a websites 404.

Example usage : python autoFuzz.py -bnf -u "http://exampleusageurl.com/"

If you want to look at the code, I will be including it at the end of this.

Results so far

So far I have only done limited testing on local instances since its not a wise idea to just throw weird strings at peoples web applications.
